# Product Requirements Document - Agent Assist Plugin (Phase 1)

**Author:** Master  
**Date:** 2026-03-16  
**Status:** Draft

## 1. Project Overview
The "Agent Assist" project integrates an AI-powered plugin directly into the Unibox Text editor. This plugin connects to a backend AI model (e.g., ChatGPT or a similar LLM) that is **strictly grounded in Knowledge Bases (KBs) ingested via the Chatbot/NextIQ KB ingestion service API**. Operations or knowledge teams upload URLs and documents as KBs through that API, and Agent Assist uses only those curated KBs to answer questions and help agents compose responses.

When agents are handling customer issues, the plugin acts as a built-in **search and text-manipulation tool** inside Unibox. Phase 1 focuses on core functional capabilities, integration with the KB ingestion and AI layers, and strong OOS controls, while **deliberately avoiding major UI redesigns** of Unibox.

## 2. Goals & Objectives
- **Efficiency:** Reduce average handling time (AHT) for support agents by enabling fast KB-grounded answers inside Unibox.  
- **Accuracy:** Ensure all AI-generated answers are strictly sourced from **verified company KBs ingested via Chatbot/NextIQ**.  
- **Safety / OOS:** Enforce hard **Out-of-Scope (OOS)** behavior whenever the KB cannot answer, avoiding hallucinated responses.  
- **Quality:** Provide integrated tools to rapidly improve the tone, length, and clarity of agent responses.

## 3. Target Audience
- **Primary Users:** Customer Support/Service Agents working within the Unibox Text editor.

## 4. Scope & Capabilities (Phase 1)

### 4.1. Core Features
1. **Contextual Search & QA (KB-Grounded):**
   - The plugin must allow agents to search seamlessly for solutions to customer issues within the editor.  
   - All answers must be generated by the AI using **only KBs that were ingested via the Chatbot/NextIQ KB ingestion service** for the relevant tenant/queue.  
   - When confidence or KB coverage is insufficient, the system must return an OOS response instead of fabricating an answer.

2. **Text Manipulation Tools (Inline Actions):**
   - **Rephrase:** Alter the phrasing of the selected text for better tone or alternative wording.
   - **Shorten:** Condense selected text while preserving the core message.
   - **Make it Larger (Expand):** Elaborate on selected shorthand or brief notes to generate a comprehensive response.
   - **Translate:** Translate the selected text into the customer's desired language.
   - **Summarize:** Provide a brief summary of long customer queries or historical chat transcripts.
   - **Suggest Response:** Auto-generate a continuous, context-aware reply based on the ongoing conversation.

### 4.2. Functional Constraints & Safeguards
- **Strict Out of Scope (OOS) Enforcement:**
   - Any query asked that falls beyond the ingested Knowledge Bases (KBs) must explicitly return an **"Out of Scope" (OOS)** or **"I don't have this information"** style message.  
   - The system *shall not* hallucinate answers using public datasets or generic internet knowledge outside of the approved KBs.  
   - The backend should return a structured OOS signal that the Unibox plugin can render clearly to the agent.

### 4.3. KB Ingestion via Chatbot/NextIQ
- The solution shall rely on the **Chatbot/NextIQ KB ingestion service API** to ingest and update KB content (URLs and documents).  
- Each KB shall be associated with the correct tenant / product / queue so that Agent Assist queries the right KB set for each interaction.  
- Re-ingestion or updates shall follow the semantics of the ingestion API (e.g., upsert vs replace), ensuring that answers reflect the latest approved content.  
- KB identifiers returned by the ingestion service shall be stored/accessible so the AI backend can route retrieval to the correct KB index.

## 5. UI/UX Requirements
- **Minimal Changes:** Because this is Phase 1, UI updates will be limited to lightweight additions (e.g., icon, side panel trigger, or context menu options).  
- **Integration point:** The plugin should be accessible via a minimal interface within the existing Unibox Text editor without requiring a redesign of the editor page.  
- **State Feedback:** The UI should clearly indicate when a result is OOS, when no KB is configured, and when backend errors occur.

## 6. Assumptions & Dependencies
- Backend AI services (like OpenAI API/Azure) are available and compliant with data security requirements.  
- The **Chatbot/NextIQ KB ingestion service API** is available, stable, and able to ingest URLs/documents and expose KB identifiers for querying.  
- The Knowledge Bases used for Retrieval-Augmented Generation (RAG) are regularly updated via the ingestion service and accessible via an index.

## 7. Architecture & Technical Dependencies

### 7.1. Architectural Diagram

The architecture follows a standard backend-for-frontend (BFF) pattern connecting the existing Unibox Text Editor with an AI Middle Layer, and finally the LLM Service and Knowledge Base (Vector Database).

### 7.2. Technical Dependencies

1. **Frontend Integration:**
   - Unibox Editor Extension API or DOM injection capabilities (depending on Unibox architecture).
   - Component library for the plugin UI (e.g., React/Vue components matching Unibox styling).

2. **Backend / Middleware:**
   - A dedicated AI API Gateway to handle request routing, prompt formatting, KB selection, and interaction logging.  
   - Authentication system capable of verifying the active Agent session (e.g., OAuth2, JWT).

3. **Data & AI Infrastructure:**
   - **LLM Provider:** API access to a secure Enterprise LLM (e.g., Azure OpenAI) fulfilling regional compliance.  
   - **Vector Database / Search:** A search indexing system (e.g., Pinecone, Weaviate, or Azure AI Search) populated exclusively with KB content ingested via the Chatbot/NextIQ KB ingestion service.  
   - **Ingestion / ETL:** Use the **Chatbot/NextIQ KB ingestion API** as the primary mechanism to ingest and update KB content from URLs/documents, avoiding ad-hoc or parallel ingestion pipelines where possible.

## 8. Future Phases (Out of Scope for Phase 1)
- Advanced analytics dashboards for plugin usage.
- Major UI/UX overhauls for the Unibox editor.
- Automated sending of messages without agent review (Human-in-the-loop is mandatory for Phase 1).
